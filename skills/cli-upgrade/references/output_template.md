# 输出模板

## 结构

```
# {tool} CLI 版本更新对比

> **最新稳定版：v{latest}**（{date} 发布）

## 一句话总览
最近几个版本主要在 XXX 方面下了功夫，同时新增了 YYY 等实用功能。

## v{latest}（最新 · {date}）

| 类别 | 你能感知到的变化 |
|------|-----------------|
| **新功能** | 一句话描述，通俗易懂 |
| **体验优化** | 一句话描述 |
| **重要修复** | 一句话描述 |
| **性能提升** | 一句话描述（如有） |

（每个版本重复表格，共 4 个版本）

## 升级方式

\`\`\`bash
# 具体安装/升级命令
\`\`\`

Sources:
- [Release vX.Y.Z](url)
- ...
```

## 规则

1. 每个版本表格控制在 2-4 行，只挑用户能直观感知的变化
2. 保持大白话、短句、避免术语堆叠
3. 类别灵活选取：新功能 / 体验优化 / 重要修复 / 性能提升 / 安全修复 / 模型更新 / VS Code 等
4. 同类别多条变化可合并为一行，用分号分隔
5. 一句话总览要概括 4 个版本的整体趋势，不要只说最新版
6. Sources 必须附上每个版本的 GitHub Release 链接
