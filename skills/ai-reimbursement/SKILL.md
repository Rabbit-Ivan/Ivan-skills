# AI 报销助手

一句话触发，自动生成全套 AI 工具订阅报销材料：审批文案（报销说明.md）+ 暗黑玻璃态 HTML 信息图（index.html）。

## 工作流程

### Step 1: 解析需求

从用户输入中提取：
- **工具名称**：如 ChatGPT Plus、Claude Pro、Cursor Pro、Lovart Ultimate
- **金额**（美元）：如 $200/月、$20/月
- **报销类型**：续费 / 新订阅 / 升级

如果信息不完整，直接向用户询问缺失项。

### Step 2: 收集/确认用户信息

检查当前对话、项目资料或用户已提供的历史信息中是否已有报销基本信息（姓名、部门、职位、项目名称）：

- **有记录**：展示已有信息，问"信息有变化吗？直接回车即跳过"
- **无记录**：逐项向用户收集。除非用户明确要求长期保存，否则只用于本次报销材料生成

需要收集的信息：
- 姓名
- 部门
- 职位
- 当前主要项目名称

### Step 3: 收集补充材料

询问用户是否有以下材料（均可跳过）：
- 订阅页面 / 到期提醒截图
- 官网功能截图
- 发票 / 支付凭证（PDF 或截图）

用户提供文件路径后，用终端命令验证文件存在。

### Step 4: 联网搜索（必须执行）

联网搜索以下内容（竞品从产品知识库中获取）：

1. `{工具名} latest update features {当前年月}` — 最新版本动态
2. `{工具名} vs {竞品1} vs {竞品2} comparison {当前年}` — 竞品对比
3. `{工具名} pricing plan {当前年}` — 最新价格

**搜索失败处理**：
- 明确告知用户哪些信息未能获取
- 使用产品知识库兜底
- 在信息图中标注哪些数据来自搜索、哪些来自兜底

### Step 5: 生成全套产出

1. **创建文件夹**：在用户当前工作目录下创建 `MMDD-工具名/`（MMDD = 当天月日）
2. **生成审批文案**：写入 `报销说明.md`
3. **生成 HTML 信息图**：读取 `references/html-template.md` 获取模板骨架，填充内容后写入 `index.html`
4. **归档截图/发票**：将用户提供的文件复制到文件夹
5. **输出完成报告**：列出所有文件及其用途

## 产出物规格

### 文件夹命名

`MMDD-工具名/`，示例：`0330-ChatGPT Pro/`、`0415-Claude Max/`

### 报销说明.md

纯文本，可直接粘贴到企业审批系统。500-800 字。

**内容结构**：

```
# {工具名} 报销说明

## 一、工具介绍
{工具名称、版本、核心能力，1-2 段}

## 二、工作场景
{在用户日常工作和项目中的具体用途，2-3 段，结合用户的项目名称}

## 三、费用明细
- 订阅套餐：{套餐名}
- 费用金额：${美元金额}/月（约 ¥{人民币金额}）
- 订阅周期：{起止日期}
- 备注：人民币金额仅供参考，请以实际付款汇率为准

## 四、续费必要性
{效率提升论证、最新版本亮点、不续费的影响，1-2 段}
```

### index.html（暗黑玻璃态信息图）

生成前**必须先读取** `references/html-template.md`，获取完整的 HTML/CSS 骨架和样式规范。

HTML 包含以下 6 个模块（垂直滚动布局）：

| # | 模块 | 内容要点 | 数据来源 |
|---|------|---------|---------|
| 1 | 头部摘要 | 工具名+版本、金额（$+¥）、申请人+部门、日期 | 用户输入 |
| 2 | 为什么需要 | 核心问题、工作场景应用、效率提升→反哺团队 | 用户信息+搜索 |
| 3 | 竞品对比 | Chart.js 雷达图/柱状图、2-3 竞品、多维度 1-10 分 | 搜索+知识库 |
| 4 | 最新功能 | 2-4 个新功能亮点、卡片式布局 | 联网搜索 |
| 5 | ROI 分析 | 使用前后对比、日均成本、Chart.js 柱状图 | 综合评估 |
| 6 | 截图凭证 | Base64 内嵌截图、卡片式展示 | 用户提供 |

- 模块 6 仅在用户提供了截图/发票时生成
- 竞品对比图表下方注明"数据基于公开评测和产品对比综合评估"

## 汇率处理

- 用户提供美元金额（所有 AI 工具均以美元计价）
- 用近期大致汇率估算人民币金额（如 1 USD ≈ 7.2-7.3 RMB）
- 审批文案和 HTML 中均标注"仅供参考，请以实际付款汇率为准"

## 图片 Base64 内嵌

用终端命令将用户截图转为 Base64，直接嵌入 HTML（不压缩）：

```bash
base64 -i screenshot.png | tr -d '\n'
```

HTML 中使用：
```html
<img src="data:image/{mime};base64,{encoded}" class="..." />
```

MIME 映射：`.png` → `image/png`、`.jpg`/`.jpeg` → `image/jpeg`、`.gif` → `image/gif`、`.webp` → `image/webp`、`.pdf` → 不内嵌，直接复制文件到文件夹

## 边界约束

1. 只生成材料，不自动提交审批
2. 只处理 AI 工具订阅类报销，不处理差旅/餐饮/办公用品
3. 不存储信用卡号等敏感财务信息
4. 每次必须执行联网搜索
5. 搜索失败必须明确告知用户缺失了什么
