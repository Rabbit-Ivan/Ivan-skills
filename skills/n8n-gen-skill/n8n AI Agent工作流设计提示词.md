# n8n AI Agent 工作流设计提示词

## 设计目标

- 先保证可导入、可运行，再做功能扩展。
- 节点职责单一，分支逻辑清晰，避免无意义复杂度。
- 所有外部依赖（API Key、Webhook URL、数据库连接）必须参数化，不写死凭据。

## 架构原则

1. 触发器明确：优先使用 `Webhook`、`Cron`、`Manual Trigger` 之一作为入口。
2. 数据标准化：在进入核心业务节点前统一字段格式。
3. 错误隔离：对高风险节点添加 `Error Trigger` 或错误分支。
4. 可观测性：关键路径保留日志字段（输入摘要、执行结果、错误信息）。
5. 幂等性：涉及写操作时加入去重键或状态判断，避免重复执行。

## AI Agent 节点建议

- 模型节点前先做上下文裁剪，控制 token 成本。
- 提示词结构固定为：角色、目标、输入、约束、输出格式。
- 输出优先 JSON（可被后续节点解析），避免自由文本。
- 对模型输出增加结构校验，不通过则进入重试或降级分支。

## Loop / Merge 设计建议

- `Split In Batches` 仅用于可独立处理的数据单元。
- `Merge` 前确保所有分支字段命名一致。
- 合并冲突字段时显式重命名，避免隐式覆盖。

## 参数建议（可按项目调整）

- temperature: `0.2`（稳定输出）
- max tokens: 按任务规模控制，默认 `1024`
- 重试次数: `2`
- 重试退避: `2s -> 4s`

## 交付要求

- 输出 JSON 必须为合法 n8n workflow 格式。
- 生成配套 `requirements.md`，包含：目标、节点清单、数据流、异常处理、部署说明。
- 在文档中列出需要用户补充的凭据与环境变量。
