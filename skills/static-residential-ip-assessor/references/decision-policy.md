# Decision Policy（Conservative）

适用于账号安全优先场景（注册登录、支付、账号养号、高价值账号操作）。

## 1) 先看“红线条件”

命中任一条，直接 `Reject`：

1. `Abuse / Block Risk >= 80`
2. IPQS 显示 `proxy=true` 且 `vpn=true`
3. 明确 `hosting=true` 且 ASN/归属为云厂商或托管商
4. 多源一致标记为高风险代理节点

## 2) 再看“灰区条件”

命中任一条，标记 `Review`（人工复核/二次验证）：

1. `Abuse / Block Risk` 在 `60-79`
2. 静态家宽属性分 `40-69`
3. 数据源冲突明显，且关键证据缺失（如缺少 BGP 或缺少 IPQS）
4. 近期（7 天内）风险分波动大
5. IPQS 数据不可用或结果无法验证

## 2.1 Review 放行规则（已启用）

`Review` 不等于永久拒绝，满足二次验证后可放行：

1. 二次验证通过（至少满足 2 项）：
- 人机验证通过（CAPTCHA 或等效机制）
- 账号行为连续 3-7 天无异常告警
- 同设备/同地理行为一致性良好
- 补充数据源复查后风险分下降到 `< 60`

2. 不得命中红线条件（第 1 节任一条）

3. 放行范围限制：
- 先放行低敏感操作，再逐步放行高敏感操作（支付最后放行）

## 3) 放行条件

同时满足以下条件才 `Allow`：

1. `Static Residential Confidence >= 70`
2. `Abuse / Block Risk <= 39`
3. 未命中 proxy/vpn/hosting 关键风险标签
4. 证据完整度高（核心来源齐全且时效 <= 7 天）
5. IPQS 可用且结果可审计（包含观测时间与关键字段）

## 4) 场景加严系数

同一 IP 在不同场景按以下方式加严：

1. 支付/金融：风险分 +10 后再决策
2. 账号养号：风险分 +8 后再决策
3. 注册登录：风险分 +5 后再决策
4. 一般账号安全巡检：不加分

## 5) 对 Claude 账号安全的特殊建议

1. 若处于 `Review`，先完成二次验证再用于关键账号操作。
2. 若过去 14 天未复查，先做复查再继续使用。
3. 若同段出现多个高风险样本，整段降级处理。

## 6) CIDR 灰度连坐策略（半开）

不启用“全段拉黑”，只启用“阈值触发 + 降级”：

1. 触发阈值（满足其一）：
- 同一 CIDR 在 14 天内出现 `>= 3` 个高风险样本（风险分 >= 70）
- 同一 CIDR 抽样命中率 `>= 20%`

2. 触发后动作：
- 该段默认动作降级为 `Review`，不是 `Reject`
- 仅对高敏感场景（支付、账号安全关键操作）加严
- 常规场景保留二次验证后放行通道

3. 自动恢复：
- 连续 14 天未新增高风险样本，则解除段级降级

## 7) 输出格式要求

报告必须包含：

- 最终动作：`Allow` / `Review` / `Reject`
- 动作原因（最多 3 条关键依据）
- 可执行下一步（例如：更换 IP、二次验证、复查时间）
