# Scoring Rubric

使用双分数模型，避免“属性判断”和“风控风险”混在一起。

## 1) Static Residential Confidence（0-100）

含义：目标是否为“静态家宽住宅”。

### 评分维度与权重

1. ASN 与运营商属性（0-25）
- 住宅 ISP 明确：20-25
- 混合/不明确：8-19
- 云/托管/机房明显：0-7

2. BGP 与前缀一致性（0-15）
- RDAP 持有方与 BGP 起源可解释一致：10-15
- 可解释但存在较大跳变：4-9
- 明显矛盾且不可解释：0-3

3. PTR 与相邻命名稳定性（0-20）
- 有稳定住宅样式且相邻一致：14-20
- PTR 缺失或模式弱：6-13
- 明显云/机房命名：0-5

4. hosting/proxy/mobile 标记（0-15）
- 全部否：10-15
- 存在单项可疑：4-9
- 多项为真：0-3

5. 段级声誉与代理渗透（0-15）
- 段信誉稳定：10-15
- 存在代理渗透但未命中目标：4-9
- 段被广泛标注代理：0-3

6. 数据完整性与时效（0-10）
- 多源齐全且 7 天内：8-10
- 部分缺失或超过 30 天：3-7
- 缺失严重：0-2

### 结论阈值

- 80-100：高可信静态家宽住宅
- 60-79：可能为家宽，但静态性不足或证据冲突
- 40-59：属性不稳，偏商宽/混合出口
- 0-39：高概率非家宽住宅

## 2) Abuse / Block Risk（0-100）

含义：目标在风控系统中被拦截或高审查的概率。

### 评分维度与权重

1. IPQS Fraud Score 归一（0-40）
- 直接映射：`score * 0.4`

2. IPQS 代理标签（0-25）
- `proxy=true` 或 `vpn=true`：15-25
- 全否：0-5

3. AbuseIPDB 与历史举报（0-20）
- 高置信滥用：12-20
- 低/无举报：0-6

4. 多源一致性放大（0-15）
- 多源一致高风险：10-15
- 单源高风险：4-9
- 多源一致低风险：0-3

### 风险阈值

- 0-39：低风险
- 40-69：中风险
- 70-89：高风险
- 90-100：极高风险

## 3) 冲突处理规则

1. 不互相覆盖
- “静态家宽属性高”与“风控风险高”可以同时成立。

2. 明确证据强弱
- 属性优先看 ASN/BGP/RDAP。
- 风控优先看 IPQS 与黑名单体系。

3. 输出并存结论
- 必须写出：为什么两类结论并存，以及业务上该如何用。

## 4) 保守模式覆盖规则

当用户强调账号安全优先时，启用以下覆盖逻辑：

1. 若 `Abuse / Block Risk >= 70`，即使属性分高，也不得直接 `Allow`。
2. 若命中 `proxy=true` 或 `vpn=true`，最低动作为 `Review`。
3. 若同时命中 `proxy=true` 与 `vpn=true`，默认 `Reject`。
4. 若证据缺失超过 2 个核心来源（RDAP/BGP/IPQS/AbuseIPDB），默认 `Review`。
5. 若 IPQS 缺失，则 `Abuse / Block Risk` 结果仅可标记为“暂定”，且最终动作不得高于 `Review`。
