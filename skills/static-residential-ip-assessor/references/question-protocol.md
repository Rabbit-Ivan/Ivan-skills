# Question Protocol（Socratic）

在开始调查前，先用最少问题锁定目标，减少误判。

## Round 1: 先问目标，不先问细节

每轮最多 3 个问题，优先级如下：

1. 你最看重哪件事？
- A. 判断是否静态家宽住宅
- B. 判断是否会被风控拦截
- C. 两者都要

2. 你的使用场景是什么？
- 登录/注册、支付、广告投放、账号养号、其他

3. 你给的是哪种输入？
- 单个 IP
- IP 段（CIDR）
- ASN + Hostname + Range

## Round 2: 决定“严格度”

仅在需要时继续问：

1. 你希望误判成本偏向哪边？
- 宁可误杀（严格）
- 宁可放过（宽松）
- 平衡

2. 你是否接受“来源冲突但可解释”的结论？
- 接受：输出并存结论 + 建议
- 不接受：必须给单一结论（会降低可信度）

3. 你是否需要可审计证据？
- 需要：每条结论必须带来源与时间
- 不需要：可给简版摘要

## Stop Rule

满足以下条件后停止提问并开始分析：

- 已明确目标（属性/风险/双目标）
- 已明确输入类型
- 已明确严格度偏好

如果用户没有回复，采用默认值：

- 目标：双目标（属性 + 风险）
- 严格度：保守
- 输出：可审计报告（含来源和时间戳）

## Round 3: 高价值账号补充问题（可选）

当用户提到账号安全、支付、Claude 封号风险时，再问：

1. 是否启用“保守决策策略”？
- 是：按 `decision-policy.md` 输出 Allow/Review/Reject
- 否：只输出分数和建议，不给硬决策

2. 是否允许“灰区放行”？
- 不允许：Review 一律不放行
- 允许：Review 可在二次验证后放行

3. 复查周期偏好？
- 每 7 天
- 每 14 天
- 每 30 天

4. CIDR 连坐策略偏好？
- 全开：命中阈值后整段高强度拦截（误伤高，不推荐）
- 半开灰度：命中阈值后整段降级为 Review（推荐）
- 关闭：不做段级联动（漏判风险更高）

## 高价值账号默认值（当用户明确账号安全优先）

- 灰区放行：允许（需二次验证）
- 复查周期：每 14 天
- CIDR 策略：半开灰度（阈值触发 + 降级，不全段拉黑）
